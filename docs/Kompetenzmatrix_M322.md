# Modul 322 - Benutzerschnittstellen entwerfen und implementieren - Kompetenzmatrix

| **Kompetenzband** | **HZ** | **Beginner** | **Intermediate** | **Advanced** |
|------------------|---------------|----------------|--------------------|----------------|
| A - Anforderungsanalyse verstehen | 1 | A1B - Ich kann die Bestandteile einer Anforderungsanalyse nennen (z.B. User, Aufgaben, Kontext und Technologie) | A1I - Ich kann die Ergebnisse einer Anforderungsanalyse erläutern (z.B. User, Aufgaben, Kontext und Technologie). | A1A - Ich kann eine bestehende Anforderungsanalyse hinterfragen und Verbesserungen vorschlagen. |
| A - Vorgehensmodell wählen | 1 | A2B - Ich kann ein geeignetes Vorgehensmodell zur Erhebung von Anforderung nennen und seine Phasen aufzählen (z.B. Design Thinking, User Centered Design). | A2I - Ich kann den Sinn und Zweck eines Vorgehensmodells erklären und seine Phasen erläutern (z.B. Design Thinking, User Centered Design). | A2A - Ich kann ein iteratives Vorgehensmodell begründet auswählen. |
| A - Nutzungskontext verstehen | 1 | A3B - Ich kann eine Methode zur Analyse von Nutzungsumfeld, Aufgaben und Benutzerverhalten erklären (z.B. Beobachtung, Interview, Loganalyse). | A3I - Ich kann eine Methode zur Analyse von Nutzungsumfeld, Aufgaben und Benutzerverhalten anwenden (z.B. Beobachtung, Interview, Loganalyse). | A3A - Ich kann die gewonnen Erkenntnisse aus der Anwendung einer Methode sinnvoll in Anforderungen überführen. |
| A - Benutzereigenschaften erfassen | 1 | A4B - Ich kann den Zweck einer methodischen Erfassung von Benutzereigenschaften, Zielen und Bedürfnissen begründen. | A4I - Ich kann aus Benutzereigenschaften, Zielen und Bedürfnissen die wichtigsten Benutzerprofile ableiten und methodisch erfassen (z.B. Persona, Empathy Map). | A4A - Ich kann Benutzereigenschaften, Ziele und Bedürfnisse hinterfragen und Benutzerprofile dahingehend optimieren. |
| A - Nutzungsanforderungen spezifizieren | 1 | A5B - Ich kann eine Methode zur Dokumentation und Spezifikation von Nutzungsanforderungen erklären (z.B. Use Cases, User Stories). | A5I - Ich kann eine Methode zur Dokumentation und Spezifikation von Nutzungsanforderungen anwenden (z.B. Use Cases, User Stories). | A5A - Ich kann eine Methode zur Bewertung und Priorisierung von Nutzungsanforderungen anwenden (z.B. Backlog). |
| B - Grundsätze der Dialoggestaltung verstehen | 2 | B1B - Ich kann das Konzept der Gebrauchstauglichkeit (Usability) erklären (Effektivität, Effizienz, Zufriedenstellung). | B1I - Ich kann die Interaktionsprinzipien einer bestehenden Benutzerschnittstelle erklären (z.B. nach ISO 9241-110). | B1A - Ich kann Interaktionsprinzipien für eine geforderte Benutzerschnittstelle festlegen. |
| B - Benutzerschnittstelle entwerfen | 2 | B2B - Ich kann einzelne Dialoge (Screens, z.B. als Wireframe) skizzieren. | B2I - Ich kann eine Abfolge (Flow) von Dialogen (Screens) als Papierprototyp präsentieren. | B2A - Ich kann eine Abfolge (Flow) einer Benutzerschnittstelle als Klickprototypen entwerfen. |
| B - Interaktionsprinzipien anwenden | 2 | B3B - Ich kann verschiedene Interaktionselemente einer bestehenden Benutzerschnittstelle identifizieren (z.B. Menu, Navigation, Orientierung, Ordnung). | B3I - Ich kann für eine Benutzerschnittstelle passende Interaktionselemente nach entsprechenden Konventionen auswählen und sinnvoll anordnen (z.B. semantischer Aufbau, Darstellung, Benennung, Struktur). | B3A - Ich kann eine bestehende Benutzerschnittstelle in Bezug auf die Interaktionselemente analysieren und optimieren. |
| B - Eingabeformate kennzeichnen | 2 | B4B - Ich kann Regeln zur Kennzeichnung von Pflichtfeldern nennen (z.B. Sternchen). | B4I - Ich kann Regeln zur Kennzeichnung von erwarteten Eingabeformaten nennen (z.B. Datum, E-Mail, Telefon). | B4A - Ich kann Pflichtfelder und erwartete Eingabeformate einer Benutzerschnittstelle selbsterklärend kennzeichnen. |
| B - Hilfe und Feedback integrieren | 2 | B5B - Ich kann geeignete Hilfefunktionen für eine Benutzerschnittstelle identifizieren und vorschlagen. | B5I - Ich kann geeignete Feedbackinformationen für eine Benutzerschnittstelle identifizieren und vorschlagen. | B5A - Ich kann sämtliche Hilfefunktionen und Feedbackinformationen einer Benutzerschnittstelle abbilden. |
| C - Elemente einer Benutzerschnittstelle kennen und anwenden | 3 | C1B - Ich kenne einfache Controls und Widgets und deren Eigenschaften für Benutzerschnittstellen und kann diese benennen und erklären. | C1I - Ich kann verschiedene Controls und Widgets für ein Interaktionsziel auswählen und sinnvoll platzieren. | C1A - Ich kann komplexere und zusammengesetzte Interaktionselemente entwerfen und in eine logische Abfolge bringen. |
| C - Benutzerschnittstelle erstellen und testen | 3 | C2B - Ich setze die entworfenen Skizzen in einen klickbaren Prototyp um. | C2I - Ich kann ein Vorhaben mit einem klickbaren Prototyp testen und mögliche Probleme identifizieren. | C2A - Ich kann Probleme analysieren, Verbesserungsvorschläge anbringen und den Prototypen überarbeiten. |
| C - Benutzerfreundlichkeit verbessern | 3 | C3B - Bezüglich der Benutzerfreundlichkeit erkenne ich schwierig umzusetzende Elemente. | C3I - Ich kann Vorschläge zur Verbesserung Benutzerfreundlichkeit unterbreiten. | C3A - Ich kann aus einem Element bezüglich Benutzbarkeit eine bessere, einfachere und intuitive Lösung umsetzen (erarbeiten). |
| D - Usability testen | 4 | D1B - Ich kann einen Walkthrough einer Benutzerschnittstellen-Abfolge mit jemandem durchführen. | D1I - Ich kann einen Usability-Test für eine Abfolge von Benutzerschnittstellen bezüglich Benutzerfreundlichkeit durchführen. | D1A - Ich kann einen Usability-Test für eine Abfolge von Benutzerschnittstellen bezüglich Benutzerfreundlichkeit und auch Business-Effizienz planen und durchführen. |
| D - Usability quantifizieren und verbessern | 4 | D2B - Ich kann einen standardisierten Fragebogen (SUS, HEART) erklären. | D2I - Ich kann anhand des Fragebogens (SUS, HEART) eine Nachbefragung durchführen und auswerten. | D2A - Ich kann Ergebnisse (Metrik) analysieren und Verbesserungsvorschläge anbringen. |
| E - Barrierefreiheit umsetzen | 5 | E1B - Ich kann die Anforderungen an eine barrierefreie Benutzerschnittstelle nennen und kann erklären worauf es beim Entwurf einer Anwendung ankommt. | E1I - Ich kann Elemente einer Benutzerschnittstelle barrierefrei umsetzen (Labels, Tab-Navigation, Kontrast, Alternativtext usw.). | E1A - Ich kann geeignete Elemente für eine barrierefreie Benutzerschnittstelle vorschlagen. |
| E - Barrierefreiheit prüfen | 5 | E2B - Ich kann die Möglichkeiten zur Überprüfung der Barrierefreiheit erläutern (z.B. Checkliste). | E2I - Ich kann eine Benutzerschnittstelle gemäss Checkliste auf Barrierefreiheit prüfen. | E2A - Ich kann die Barrierefreiheit einer Benutzerschnittstelle nachweisen. |

## Legende
HZ = Handlungsziele
1. Anforderungen an Benutzerschnittstellen erheben und spezifizieren.
2. Benutzerschnittstellen entwerfen.
3. Benutzerschnittstellen erstellen und testen.
4. Usability von Benutzerschnittstellen testen und verbessern.
5. Barrierefreiheit von Benutzerschnittstellen umsetzen und prüfen.
